

# Product Requirements Prompt: Neo-Brutalist Sidebar Overhaul

**Role:** Expert CSS Engineer for Streamlit
**Objective:** The main content area is now successfully Neo-Brutalist, but the sidebar remains default Streamlit (gray, rounded, soft). The goal is to apply the same **hard-edged, high-contrast** design to the sidebar elements.

## 1. CSS Selectors (The "Hard" Part)

You must inject a new CSS block that targets the specific sidebar classes. Do not just apply global styles; they won't penetrate the sidebar widgets effectively.

### A. Sidebar Container

* **Selector:** `[data-testid="stSidebar"]`
* **Style:**
* `background-color: #FFC900;` (Electric Yellow) OR `background-color: #FFFFFF;` with a thick right border.
* `border-right: 3px solid #000000;`



### B. Sidebar Widgets (Dropdowns & Inputs)

Streamlit uses `baseweb` for dropdowns, which are notoriously hard to style. We need to force them.

* **Target:** `[data-testid="stSidebar"] div[data-baseweb="select"] > div` (The Dropdown Box)
* **Style:**
* `border: 2px solid #000000 !important;`
* `border-radius: 0px !important;`
* `background-color: #FFFFFF !important;`
* `box-shadow: 4px 4px 0px 0px #000000 !important;`
* `color: black !important;`



### C. Sidebar Typography

* **Target:** `[data-testid="stSidebar"] h1, [data-testid="stSidebar"] h2, [data-testid="stSidebar"] h3, [data-testid="stSidebar"] label`
* **Style:**
* `font-family: 'Space Grotesk', monospace !important;`
* `text-transform: uppercase;`
* `font-weight: 800;`
* `color: #000000 !important;`



## 2. Code Implementation

Append this specific block to your existing CSS injection in `app.py`:

```python
st.markdown("""
    <style>
    /* 1. SIDEBAR BACKGROUND */
    [data-testid="stSidebar"] {
        background-color: #FFFFFF;
        border-right: 3px solid black;
    }
    
    /* 2. SIDEBAR HEADERS */
    [data-testid="stSidebar"] h1, 
    [data-testid="stSidebar"] h2, 
    [data-testid="stSidebar"] h3, 
    [data-testid="stSidebar"] div[data-testid="stMarkdown"] p {
        font-family: 'Space Grotesk', monospace;
        text-transform: uppercase;
        font-weight: bold;
        color: black;
    }

    /* 3. SIDEBAR INPUTS (Text & Number) */
    [data-testid="stSidebar"] input {
        border: 2px solid black;
        border-radius: 0px;
        box-shadow: 3px 3px 0px 0px black;
    }

    /* 4. SIDEBAR DROPDOWNS (The Tricky Part) */
    /* This targets the container of the select box */
    div[data-baseweb="select"] > div {
        background-color: white !important;
        border: 2px solid black !important;
        border-radius: 0px !important;
        box-shadow: 3px 3px 0px 0px black !important;
        color: black !important;
    }
    
    /* Force text inside dropdowns to be black */
    div[data-baseweb="select"] span {
        color: black !important;
    }
    
    /* 5. SIDEBAR BUTTONS (If any) */
    [data-testid="stSidebar"] button {
        background-color: #FF90E8;
        border: 2px solid black;
        box-shadow: 3px 3px 0px 0px black;
        border-radius: 0px;
    }
    </style>
""", unsafe_allow_html=True)

```

### One Final Design Note

If you want to go full Gumroad/Brutalist, consider **removing the sidebar entirely** and putting those settings in an "Expandable" block (`st.expander`) under the main Start button.

Why?

1. **Mobile Friendly:** Sidebars are clunky on phones.
2. **Aesthetic:** An `st.expander("⚙️ CONFIGURE MISSION PARAMETERS")` that drops down with a hard black border looks much cooler than the persistent gray sidebar.
