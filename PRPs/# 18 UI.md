
# Product Requirements Prompt: The "Modern SaaS" Overhaul (Card UI)

**Role:** Senior Frontend Designer for Streamlit
**Objective:** Completely break the "default Streamlit document" look. Implement a **Card-Based Layout** where the main interface floats on a soft gray background, similar to a modern login page or dashboard (e.g., Linear, Vercel).

## 1. Design System Updates (CSS Injection)

You must inject the following CSS logic using `st.markdown(..., unsafe_allow_html=True)` at the top of the app.

### A. Global Typography & Background

* **Import Font:** `@import url('https://fonts.googleapis.com/css2?family=Inter:wght@400;600;700&display=swap');`
* **Apply Font:** Set `html, body, [class*="css"] { font-family: 'Inter', sans-serif; }`
* **Background:** Set `.stApp { background-color: #F8F9FA; }` (A very soft, premium gray).

### B. The "Main Card" Effect (Critical)

Streamlit apps usually stretch full width. We want a focused "Card" look.

* **Target:** The main `block-container`.
* **Style:**
* `background-color: #FFFFFF;`
* `border-radius: 20px;`
* `padding: 3rem !important;`
* `box-shadow: 0px 10px 40px rgba(0,0,0,0.08);`
* `max-width: 900px;` (Constrain width to keep it readable).
* `margin: auto;` (Center it).
* `margin-top: 50px;`



### C. Button Hierarchy

* **Primary Button (Start Mission):** Keep the `#6C63FF` (Purple) color but add `box-shadow: 0px 4px 15px rgba(108, 99, 255, 0.4);` to make it glow slightly.
* **Secondary Button (Google Login):** Remove the gray background. Make it a **"Ghost Button"**: transparent background, dark text, subtle border. It should look clean, not blocky.
